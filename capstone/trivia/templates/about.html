Trivia

Trivia is a quiz web app that I built for my CS50W final project.

It pulls information from five public APIs, depending on the topics the user requests to be quizzed on, parses the responses and formats them into questions.

Trivia will keep track of users scores and caches viable API routes to reduce loading times.

APIs

I settled on using the following public APIs:
  • Art Institute of Chicago
  • Decathlon Sports API
  • REST Countries
  • Zoo Animal API
  • Movie Quote API
    • All of these are public and do not require authentication.
    Some of the records contain incomplete or inappropiate content, so the content will be check on the backend to ensure suitability before it is returned to the frontend in JSON format.

    Tech

    Trivia utilises Python/Django for it's backend, and HTML/CSS/Javascript for it's frontend
    While parts of the site are rendered using Django templates, trivia (/triviagame) is entirely reactive and served by a single .html page.

    Notes / To Do

    There are a few features and issues I plan on continuing to work on.
  • Question difficulty: Some topics, especially Art, generate questions that are relatviely difficult compared to other topics. These could be reworked, or hints can be provided
  • Multiplayer:I would like to add the option for multiple people to attempt the same quiz, or save a question set to be tried by others
  • Score / Rating tracking: Trivia currently only updates scores and ratings at the end of each game, before displaying them to the user when required. Progress over time or other visualisations of data could be displayed instead
  • Improve API selection: The Movie Quote API only holds a limited selection of quotes, many of which will not be returned due to being tagged as containing adult language.
